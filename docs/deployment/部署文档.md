# 宝宝取名网站部署文档

## 1. 部署环境规划

### 1.1 环境分层

**开发环境 (Development)**
- 用途：日常开发和功能测试
- 配置：轻量级资源配置
- 访问限制：仅开发团队内部访问
- 数据：使用模拟数据或脱敏数据

**测试环境 (Testing/QA)**
- 用途：系统测试、集成测试和验收测试
- 配置：与生产环境相似但规模较小
- 访问限制：测试团队和项目相关人员
- 数据：使用脱敏的生产数据副本

**预发布环境 (Staging)**
- 用途：最终验收和性能测试
- 配置：与生产环境完全一致
- 访问限制：项目关键人员
- 数据：使用脱敏的生产数据副本

**生产环境 (Production)**
- 用途：面向最终用户的正式服务
- 配置：高可用、高性能配置
- 访问限制：公开访问，后台管理需授权
- 数据：实际用户数据，定期备份

### 1.2 基础设施配置

**生产环境服务器配置**
- Web服务器：2+ 实例，每个4vCPU/8GB RAM
- 数据库服务器：主从架构，主库4vCPU/16GB RAM
- 缓存服务器：Redis集群，2+ 节点
- 负载均衡：云服务商提供的负载均衡服务
- CDN：全球分布的内容分发网络

## 2. 部署准备

### 2.1 环境要求

**操作系统**
- 推荐：Ubuntu 20.04 LTS或更高版本
- 最低：Ubuntu 18.04 LTS

**软件依赖**
- Node.js: v16.x或更高版本

- 出站连接：允许所有HTTP/HTTPS流量
- 入站连接：允许HTTP(80)、HTTPS(443)、SSH(22)
- 内部网络：允许服务间通信

## 3. 部署流程

### 3.1 基础环境配置

**系统更新**

```bash
# 更新系统包
sudo apt update && sudo apt upgrade -y

# 安装基础工具
sudo apt install -y curl git build-essential
```

**安装Docker**

```bash
# 安装Docker依赖
sudo apt install -y apt-transport-https ca-certificates gnupg lsb-release

# 添加Docker官方GPG密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 添加Docker仓库
echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io

# 安装Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/download/v2.5.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 将当前用户添加到docker组
sudo usermod -aG docker $USER
```

**安装Node.js**

```bash
# 安装NVM
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.1/install.sh | bash

# 加载NVM
export NVM_DIR="$HOME/.nvm"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"

# 安装Node.js
nvm install 16
nvm use 16
```

### 3.3 Nginx配置

**安装Nginx**

```bash
sudo apt install -y nginx
```

**配置HTTPS**

```bash
# 安装Certbot
sudo apt install -y certbot python3-certbot-nginx

# 获取SSL证书
sudo certbot --nginx -d babyname.example.com -d www.babyname.example.com
```

**Nginx配置文件**

```bash
sudo nano /etc/nginx/sites-available/babyname
```

配置文件内容：

```nginx
server {
    listen 443 ssl http2;
    server_name babyname.example.com www.babyname.example.com;

    ssl_certificate /etc/letsencrypt/live/babyname.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/babyname.example.com/privkey.pem;
    ssl_session_timeout 1d;
    ssl_session_cache shared:SSL:50m;
    ssl_session_tickets off;

    # 现代兼容性
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;

    # HSTS
    add_header Strict-Transport-Security "max-age=63072000" always;

    # OCSP Stapling
    ssl_stapling on;
    ssl_stapling_verify on;
    resolver 8.8.8.8 8.8.4.4 valid=300s;
    resolver_timeout 5s;

    # 安全头
    add_header X-Content-Type-Options nosniff;
    add_header X-Frame-Options DENY;
    add_header X-XSS-Protection "1; mode=block";
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' https://www.googletagmanager.com; style-src 'self' 'unsafe-inline'; img-src 'self' data: https://www.google-analytics.com; connect-src 'self' https://www.google-analytics.com; font-src 'self'; object-src 'none'; media-src 'self'; frame-src 'none'; worker-src 'self'; manifest-src 'self'";

    # 代理设置
    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态文件缓存
    location /static/ {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        expires 30d;
        add_header Cache-Control "public, max-age=2592000";
    }

    # 健康检查
    location /health {
        proxy_pass http://localhost:3000/api/health;
        proxy_set_header Host $host;
        access_log off;
    }
}

# HTTP重定向到HTTPS
server {
    listen 80;
    server_name babyname.example.com www.babyname.example.com;
    return 301 https://$host$request_uri;
}
```

**启用配置**

```bash
sudo ln -s /etc/nginx/sites-available/babyname /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

## 5. 备份与恢复

### 5.1 备份策略

**数据库备份**
- 全量备份：每日凌晨执行
- 增量备份：每6小时执行
- 事务日志：实时备份

**文件存储备份**
- 静态资源：每周备份
- 用户上传内容：每日备份

**备份存储**
- 本地存储：保留7天
- 云存储：保留90天
- 冷存储：保留1年

### 5.2 备份脚本

**MongoDB备份脚本**

```bash
#!/bin/bash
# mongodb_backup.sh

# 配置
BACKUP_DIR="/var/backups/mongodb"
DATE=$(date +%Y%m%d_%H%M%S)
MONGODB_HOST="localhost"
MONGODB_PORT="27017"
MONGODB_DB="babynames"
S3_BUCKET="s3://babynames-backup/mongodb"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 执行备份
mongodump --host $MONGODB_HOST --port $MONGODB_PORT --db $MONGODB_DB --out $BACKUP_DIR/$DATE

# 压缩备份
tar -czf $BACKUP_DIR/$DATE.tar.gz -C $BACKUP_DIR $DATE

# 上传到S3
aws s3 cp $BACKUP_DIR/$DATE.tar.gz $S3_BUCKET/

# 清理本地备份
find $BACKUP_DIR -type d -name "20*" -mtime +7 -exec rm -rf {} \;
find $BACKUP_DIR -type f -name "*.tar.gz" -mtime +7 -exec rm -f {} \;
```

### 5.3 恢复流程

**恢复时间目标 (RTO)**
- P0级故障：<1小时
- P1级故障：<4小时
- P2级故障：<24小时

**恢复点目标 (RPO)**
- 数据库：<15分钟数据丢失
- 文件存储：<24小时数据丢失

**MongoDB恢复脚本**

```bash
#!/bin/bash
# mongodb_restore.sh

# 配置
BACKUP_DIR="/var/backups/mongodb"
S3_BUCKET="s3://babynames-backup/mongodb"
MONGODB_HOST="localhost"
MONGODB_PORT="27017"
MONGODB_DB="babynames"

# 参数检查
if [ -z "$1" ]; then
  echo "Usage: $0 <backup_date>"
  echo "Example: $0 20230515_083000"
  exit 1
fi

BACKUP_DATE=$1

# 从S3下载备份
aws s3 cp $S3_BUCKET/$BACKUP_DATE.tar.gz $BACKUP_DIR/

# 解压备份
tar -xzf $BACKUP_DIR/$BACKUP_DATE.tar.gz -C $BACKUP_DIR

# 执行恢复
mongorestore --host $MONGODB_HOST --port $MONGODB_PORT --db $MONGODB_DB $BACKUP_DIR/$BACKUP_DATE/$MONGODB_DB

# 清理临时文件
rm -rf $BACKUP_DIR/$BACKUP_DATE
rm -f $BACKUP_DIR/$BACKUP_DATE.tar.gz

echo "MongoDB restore completed successfully!"
```

## 6. 扩展策略

### 6.1 水平扩展

**前端扩展**
- 触发条件：CPU利用率>70%持续5分钟
- 扩展策略：每次增加1个实例，最大10个实例
- 缩减策略：CPU利用率<30%持续10分钟，每次减少1个实例，最小2个实例

**API服务扩展**
- 触发条件：请求队列>100或响应时间>500ms
- 扩展策略：每次增加2个实例，最大20个实例
- 缩减策略：CPU利用率<40%持续15分钟，每次减少1个实例，最小3个实例

### 6.2 数据库扩展

**读写分离**
- 主库：处理所有写操作和关键读操作
- 从库：处理非关键读操作，可扩展至多个实例

**分片策略**
- 分片键：用户ID（适用于用户数据）
- 分片数量：初始2个分片，根据数据增长扩展
- 均衡器：自动平衡分片数据分布

## 7. 安全措施

### 7.1 网络安全

**安全组配置**

| 类型 | 协议 | 端口范围 | 来源 | 描述 |
|------|------|---------|------|------|
| 入站 | TCP | 22 | 办公室IP | SSH访问 |
| 入站 | TCP | 80 | 0.0.0.0/0 | HTTP |
| 入站 | TCP | 443 | 0.0.0.0/0 | HTTPS |
| 入站 | TCP | 27017 | 内部网络 | MongoDB |
| 入站 | TCP | 6379 | 内部网络 | Redis |
| 出站 | All | All | 0.0.0.0/0 | 所有流量 |

**WAF规则**

- 防止SQL注入
- 防止XSS攻击
- 防止CSRF攻击
- 限制请求速率
- 阻止恶意IP

### 7.2 数据安全

**敏感数据处理**
- 用户密码：使用bcrypt哈希存储
- API密钥：加密存储，定期轮换
- 个人信息：加密存储，访问控制

**数据库安全配置**

## 8. 故障排除

### 8.1 常见问题

**应用无法启动**

检查步骤：
1. 验证环境变量配置
2. 检查日志文件中的错误信息
3. 确认数据库连接正常
4. 验证端口是否被占用

解决方案：
```bash
# 检查环境变量
cat .env.production

# 检查日志
docker-compose -f docker-compose.production.yml logs app

# 检查数据库连接
mongo mongodb://localhost:27017/babynames

# 检查端口占用
netstat -tulpn | grep 3000
```

**数据库连接失败**

检查步骤：
1. 确认MongoDB服务运行状态
2. 验证连接字符串正确性
3. 检查网络连接和防火墙规则
4. 验证认证信息

解决方案：
```bash
# 检查MongoDB状态
docker-compose -f docker-compose.production.yml ps mongo

# 重启MongoDB
docker-compose -f docker-compose.production.yml restart mongo

# 检查日志
docker-compose -f docker-compose.production.yml logs mongo
```

**性能问题**

检查步骤：
1. 监控CPU、内存和磁盘使用情况
2. 分析应用日志中的慢请求
3. 检查数据库查询性能
4. 验证缓存有效性

解决方案：
```bash
# 检查系统资源
top

# 分析慢查询
mongo --eval "db.setProfilingLevel(2, 100)"

# 清除Redis缓存
redis-cli FLUSHALL
```

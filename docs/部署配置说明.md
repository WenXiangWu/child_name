# 部署配置说明

## 环境变量配置

### 开发环境
```bash
# .env.local
NEXT_PUBLIC_BASE_URL=http://localhost:3000
PORT=3000
```

### 生产环境配置
```bash
# .env.production
NEXT_PUBLIC_BASE_URL=https://your-domain.com
PORT=3000
```

### Vercel 部署
Vercel 会自动设置 `VERCEL_URL` 环境变量，系统会自动检测并使用。
无需手动配置 `NEXT_PUBLIC_BASE_URL`。

### Docker 部署
```dockerfile
ENV NEXT_PUBLIC_BASE_URL=https://your-domain.com
ENV PORT=3000
```

## 配置系统优势

### 1. 自动环境检测
- 客户端：自动使用 `window.location.origin`
- 服务端：优先使用环境变量，自动回退到默认值
- Vercel：自动检测 `VERCEL_URL`

### 2. 灵活的URL管理
```typescript
import { getStaticUrl, getApiUrl } from '@/lib/config';

// 自动处理不同环境的URL
const dataUrl = getStaticUrl('wuxing_dict_jianti.json');
const apiUrl = getApiUrl('generate-names');
```

### 3. 环境特定配置
```typescript
import { getEnvConfig, isDevelopment } from '@/lib/config';

const config = getEnvConfig();
// config.debug - 开发环境为true
// config.apiTimeout - 开发环境30s，生产环境10s
// config.enableCache - 开发环境false，生产环境true
```

## 故障排除

### 常见问题
1. **数据加载失败**：检查 `NEXT_PUBLIC_BASE_URL` 配置
2. **CORS 错误**：确保API和静态资源在同一域名下
3. **路径错误**：检查 `public/data/` 目录结构
